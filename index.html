Visual Embedding for Image Classification

This project is an outsourcing for a Japaneses Startup in the middle of 2018. The goal is created an application using AI and AR to increase Toy'R Us Japan's customers buying experience. Where customers can buy any items in or outside Toy'R Us store only with their smartphone.

The problem here is we need to clasify almost 2000 differents products of Toy'R Us. Due to the limitation of Computer Vision we need to collect image by ourself using mobile phone for training our AI model. So, we decided to use Triplet Loss instead of Softmax Classifier because at that time, Triplet Loss was proved strongly in Face Recognition and we will need to collect less data for training.

Our strategy here to get the best model is: Clustering the categories to group categories with similar type samples.

  • For each anchor image, we will pick 1 hardest positive, 1 hardest negative among the image batch.
  • We will keep multiple anchors and compute centroid of anchors for each category.
  • To reduce computation cost we will first match with anchor centroids.
  • If distance is lower than TL and higher than TH, we will give immediate decision that the product exists and doesn’t exist respectively.
  • We are using two threshold. Lower threshold, TL and higher threshold

In the end, we got this result, that is very good and promising.

Document Reader for Banking Transaction

The goal of this project is create a Document Reader System that can extra information from transaction documents. But not only extra from PDF, PNG, JPG to text but also the meaning of the text extracted into categories that customer wanted. So, in this Project, we have 3 stages.

At stage 1, we need to detect the location of text line. So we decided to use CTPN for text-detection. For better performance, we need to fine-tuned the pretrained model by our own dataset. Because we need to detect text-line so we have to generated our own dataset has same format as the original dataset.

At stage 2, because our customer is a big Bank so their system do not allow any internet access for privacy. So we cannot use Google OCR API or other services need internet access. So we have 2 option, use opensource like Tesseract or cooperate with some partner have OCR technology. We have a friend that is the Ph.D about OCR, so we decided to use his AI Model. Moreover, his model perform better than Tesseract.

At stage 3, because customer want to clasify the text in to their needed categories, so for the first phase, we decide to use regular expression. The downside is customer have to choose a template manually for the best classification.

We also provide a better solution, data driven approach, but this method will need more data and time for training. So it will be depended on customer expectation.

Virtual Teacher

The goal of this project is to create a Virtual Teacher that can talk, interact and validate student level. I also combine with Ohmnilabs Robot so kids to engage the interaction with kids. They can tell the Robot move around and ask him open domain questions like the weather today, who is the CEO of Apple, ...

In this project, for the chatbot engine, I used Dialogflow for fixed lesson material and Google Custom Search combine with Bert-Squad pretrained model for open domain question. For the Recognition and Speech Part, I used Mozilla DeepSpeech and Mozilla TTS. For the validation part, due to the lack of data, so right now, it just math statistics. I also re-train an emotional model to recognize student facial emotion for better analysis.
